## 2026-07-22 - Tenant enforcement threaded through every store and surface (M2 waves A-C)

**Changes:** Non-optional `tenantId` is now the first parameter of every corpus store method (Session/Fact/CodeExemplar/Signal/Workstream/Entity/Outcome stores + Source/Provider registries + the raw-SQL actions-log/overlay/dataset modules, sqlite and pg) and every corpus-returning surface (all 17 MCP tools, every FILTER HTTP route). One `tenantClause`/`tenantClausePg` helper builds every tenant WHERE fragment; a guard test scans all SQL-bearing files and fails the build on any inline `tenant_id` bind outside the helper (column-equality defense-in-depth joins explicitly permitted and restored where an earlier pass had removed them). Vector paths re-filter at id-resolution; by-id reads return not-found on tenant mismatch; `get_session`'s supersedence enrichment only surfaces log entries for tenant-resolved ids. `NLM_HOSTED=1` gates 5 whole-DB LOCAL routes and 8 shared-file routes with 403s (per-path tested). Cross-tenant leak-test contract live: 8/11 cases green on sqlite, 7/11 on pg (verified against a throwaway pgvector container); remaining cases are explicitly gated on the auth/ingest/state milestones and marked as such. Bugs fixed along the way: cross-tenant action-overlay application (entity-canonical collisions could cross-apply retire/rename/merge), un-awaited async `/api/live/*` handlers under pg, stale pg schemaVersion assertion. Suite 2225 pass / 0 fail; typecheck clean; zero local-mode behavior change (composition roots pass the default team everywhere pending the auth milestone).

**Decisions:** Actions table filters via EXISTS-joins to stamped parents (polymorphic subjects with no resolvable parent pass through - unattributable rows can't leak to any tenant). `sources.name`/`providers.name` global uniqueness re-keys to `(tenant_id, name)` in the ingest-attribution milestone, not here.

**State:** main pushed. Full serial pg suite has pre-existing order-dependent flakiness (17 failures predating this work, stale-assertion class now partially fixed) - per-file pg schema isolation is the next-milestone fix (#386-class). #408 partially answered: a throwaway `pgvector/pgvector:pg16` container works for the pg lane.

**Changes:** Threaded non-optional `tenantId: string` as the first parameter through every `SessionStore` port method (both `sqlite-session-store.ts`/`pg-session-store.ts`, plus the concrete-only `recentWrites`/`recentMarkers`/`getSessionScopeById`/`insertSession`/`listBackfillCandidates`/`insertFactsForSession`) and every `FactStore` port method (both backends, incl. `ingestSessionFactsInTxn`).

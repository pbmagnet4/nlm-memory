User: Should we track this in git so it doesn't happen again?

Agent: Yes. The issue is that git does not preserve execute bits by default on some checkout configurations. Two options: set `core.fileMode = true` in the repo's git config (the default on Linux, but can be overridden), or add an explicit `chmod` call to the deploy script so the script is always made executable after checkout.

The deploy script approach is more robust since it works regardless of git config.

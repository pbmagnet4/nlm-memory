# #352 Phase 2 + Tier-B Outcome Rollup Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the signed-off #352 Phases 2-3-TierB: derivable session metadata (persona, parent link, model/tokens, skill), deletion of the dead subagent-start stub, and the corpus-internal Tier-B outcome rollup + MCP `report_outcome` contract tool.

**Architecture:** Phase 2 adds *derivable* columns stamped at ingest and backfilled (never inert: every column has a producer on day one). Tier-B computes outcome as a derived rollup over append-only evidence (signals + supersedence + session_edges + re-derivation), never a stamped column; `unobserved` is a first-class value. Per the 2026-07-21 sign-off amendment, Tier-B leads with supersession/continuation/re-derivation; citation is computed but flagged `producer_degraded` until capture recovers.

**Tech Stack:** TypeScript hexagonal (core/ports/adapters), better-sqlite3 + Postgres parity, vitest, MCP SDK.

## Phase 3 Tier-B — outcome rollup

### Task 6: Outcome rollup core service

**Files:**
- Create: `src/core/outcome/rollup.ts`, `src/ports/outcome.ts` (types only if needed beyond shared)
- Test: `tests/unit/core/outcome/rollup.test.ts` (pure, in-memory fakes)

**Interfaces:** Produces `deriveOutcome(sessionId, deps): OutcomeVerdict` where `OutcomeVerdict = { verdict: "held" | "overturned" | "built-upon" | "re-derived-later" | "unobserved", tier: "A" | "B", confidence: "high" | "medium" | "low", evidence: string[], producerDegraded?: true }`. Evidence precedence (sign-off amendment order): (1) Tier-A signal rows for the session (strongest, from `signals` table where correlated) → (2) supersession: session superseded or its decisions revised → `overturned` (high) / un-superseded after 14+ days → `held` (medium); (3) continuation: inbound `continues` edges → `built-upon` (medium); (4) re-derivation pair membership → `re-derived-later` (medium, negative); (5) citation → included in evidence[] but NEVER sole basis for a verdict while capture is degraded: if citation is the only signal, verdict stays `unobserved` with `producerDegraded: true`. No evidence → `unobserved` (low).

- [ ] **Step 1: Failing tests** — one per verdict path incl. the citation-only → unobserved+degraded rule and signal-beats-heuristic precedence. **Step 2:** Implement pure (deps injected: sessionStore, signalReader, edgeReader). **Step 3:** Gate. **Commit** `feat(outcome): tier-B rollup service (#352 phase 3)`
